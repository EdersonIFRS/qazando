Introdução ao RestAssured

01Introdução ao RestAssured

Nesta seção, daremos início ao nosso estudo sobre RestAssured, uma biblioteca Java bastante utilizada para testar APIs RESTful. O RestAssured fornece uma sintaxe simplificada e intuitiva para testar endpoints e validar as respostas recebidas.

O que é o RestAssured?

RestAssured é uma biblioteca Java de código aberto que foi desenvolvida com o objetivo de facilitar e agilizar a automação de testes em APIs RESTful. Ela oferece uma API amigável e completa para realizar requisições HTTP, enviar parâmetros, headers, autenticações e validar as respostas recebidas.

Uma das principais vantagens do RestAssured é sua integração com o framework de testes JUnit, o que nos permite criar facilmente testes automatizados para nossas APIs e incluí-los em nossos fluxos de integração contínua.

Por que usar o RestAssured?

Existem várias razões para escolher o RestAssured como a ferramenta principal para realizar testes de API. Algumas delas incluem:

  1. Sintaxe fluente: O RestAssured utiliza uma sintaxe fluente e legível, o que torna os testes API mais claros e expressivos. Isso facilita a leitura e manutenção dos testes ao longo do tempo.

  2. Integração com JUnit: Como mencionado anteriormente, o RestAssured pode ser facilmente integrado ao JUnit, permitindo a criação de testes de API como casos de teste JUnit. Isso torna a execução e a análise dos resultados dos testes mais fácil e eficiente.

  3. Suporte para autenticação: O RestAssured inclui suporte para diversos tipos de autenticação, como Basic, Digest, OAuth, entre outros. Isso permite testar endpoints que requerem autenticação de forma simples e direta.

  4. Validação de respostas: Com a ajuda do RestAssured, podemos validar as respostas recebidas das APIs de forma fácil e robusta. Podemos verificar códigos de status, headers, conteúdo da resposta, entre outros, permitindo uma validação completa das APIs.

Testes de API com RestAssured

02Testes de API com RestAssured

Neste tópico, abordaremos o uso do RestAssured para realização de testes de API. O RestAssured é uma biblioteca Java que proporciona uma maneira simples e intuitiva de interagir com APIs RESTful e realizar assertivas nos dados retornados.

Configuração do Ambiente

Antes de iniciar os testes de API com RestAssured, é necessário configurar o ambiente de desenvolvimento. Para isso, siga os seguintes passos:

  1. Verifique se você possui o JDK (Java Development Kit) instalado em seu sistema. Caso contrário, faça o download e instalação em sua máquina;
  2. Instale uma IDE de desenvolvimento Java, como o Eclipse ou o IntelliJ IDEA;
  3. Crie um novo projeto Java na IDE;
  4. Adicione a dependência do RestAssured ao arquivo POM.xml do seu projeto, caso esteja utilizando o Maven como gerenciador de dependências:
<dependency>
  <groupId>io.rest-assured</groupId>
  <artifactId>rest-assured</artifactId>
  <version>4.4.0</version>
  <scope>test</scope>
</dependency>
  1. Verifique se todas as dependências foram devidamente baixadas e estão disponíveis no build path do projeto.

Realizando Requisições HTTP

O RestAssured permite a realização de requisições HTTP de forma simplificada e legível. Para realizar uma requisição GET a uma API, por exemplo, utilize o seguinte código:

import static io.restassured.RestAssured.*;

Response response = get("https://api.exemplo.com/users/1");

Observe que, na chamada acima, utilizamos o método estático get() para realizar a requisição GET à URL especificada. A resposta da requisição é armazenada na variável response para posterior análise.

Assertivas nos Dados

Após realizar uma requisição à API, podemos realizar assertivas nos dados retornados para garantir sua consistência. O RestAssured fornece uma sintaxe fluente para realizar essas assertivas de forma clara e concisa.

import static io.restassured.RestAssured.*;
import static org.hamcrest.Matchers.*;

Response response = get("https://api.exemplo.com/users/1");

response.then()
        .statusCode(200)
        .body("name", equalTo("John Doe"))
        .body("email", containsString("example.com"));

No exemplo acima, utilizamos o método statusCode() para verificar se o código de status da resposta é 200 (OK). Em seguida, utilizamos o método body() para verificar se o nome retornado é "John Doe" e se o e-mail contém "example.com".

Autenticação e Autorização

Em algumas situações, é necessário realizar autenticação ou autorização antes de realizar as requisições à API. O RestAssured oferece suporte para diferentes mecanismos de autenticação, como Basic Authentication, OAuth e JWT.

import static io.restassured.RestAssured.*;

given()
    .auth()
    .basic("username", "password")
when()
    .get("https://api.exemplo.com/secure-endpoint")
then()
    .statusCode(200);

No exemplo acima, utilizamos o método auth() em conjunto com o método basic() para realizar a autenticação básica com um nome de usuário e senha.

Testes de Performance

Além de realizar testes funcionais, o RestAssured também pode ser utilizado para realizar testes de performance em suas APIs. Com o auxílio da biblioteca Apache JMeter, é possível simular uma carga de usuários e avaliar o desempenho da API sob diferentes condições.

O RestAssured oferece integração com o JMeter através de sua extensão rest-assured-jmeter. Com ela, você pode criar testes de carga e stress utilizando a mesma sintaxe fluente do RestAssured.

import io.restassured.jmeter.JMeterOptions;
import io.restassured.response.Response;
import org.junit.BeforeClass;
import org.junit.Test;

import static io.restassured.RestAssured.*;
import static io.restassured.module.jmeter.RestAssuredJMeter.*;

@JMeterOptions(
    threads = 100,
    durationInSeconds = 60,
    rampUpInSeconds = 10
)
public class PerformanceTest {

    @BeforeClass
    public static void beforeClass() {
        enableJMeter();
    }

    @Test
    public void testPerformance() {
        Response response = get("https://api.exemplo.com/users");
        response.then().statusCode(200);
    }
}

No exemplo acima, configuramos o JMeter para simular uma carga de 100 threads durante 60 segundos, com ramp up de 10 segundos. Em seguida, realizamos uma requisição GET à API dentro do método de teste. Ao final, verificamos se o código de status retornado é 200 (OK).

Com essa integração, você pode realizar testes de performance de forma automatizada e analisar os resultados obtidos.

Recursos avançados do RestAssured

03Recursos avançados do RestAssured

Neste tópico do curso "Curso de RestAssured", abordaremos os recursos avançados do RestAssured, uma biblioteca Java que permite realizar testes automatizados de API.

1. Validando Respostas Complexas

O RestAssured oferece várias maneiras de validar respostas complexas de API. A seguir, veremos alguns recursos avançados para garantir que nossos testes de API sejam eficazes:

1.1. Verificação de Itens em uma Lista

Com o RestAssured, podemos facilmente verificar se um item específico está presente em uma lista retornada pela API. Isso é particularmente útil quando precisamos garantir que um determinado dado esteja presente na resposta. Veja um exemplo de como fazer isso:

List<String> responseList = given()
    .when()
    .get("/api/users")
    .then()
    .extract()
    .path("data.username");
    
assertThat(responseList, hasItem("john.doe"));

1.2. Validação de Respostas JSON Complexas

O RestAssured também permite que realizemos validações mais complexas em respostas JSON. Por exemplo, podemos verificar se uma propriedade existe, se seu valor é igual ao esperado, ou até mesmo se uma lista contém determinados itens. Veja um exemplo de como fazer isso:

given()
    .when()
    .get("/api/user/1")
    .then()
    .body("name", equalTo("John Doe"))
    .body("age", greaterThan(18))
    .body("hobbies", hasItems("reading", "coding"));

1.3. Validação de Respostas XML

Além de validar respostas JSON, o RestAssured também suporta validação de respostas XML. Assim como no caso do JSON, podemos verificar a existência de elementos, seus valores e até mesmo a ordem em que eles aparecem. Veja um exemplo de como fazer isso:

given()
    .when()
    .get("/api/book/1")
    .then()
    .body("book.title", equalTo("The Great Gatsby"))
    .body("book.author", equalTo("F. Scott Fitzgerald"))
    .body("book.price", greaterThan(0.0));

2. Autenticação e Autorização

O RestAssured também oferece recursos avançados para autenticar e autorizar solicitações em APIs protegidas. Vejamos dois métodos comumente utilizados:

2.1. Autenticação Básica

A autenticação básica é um método simples de autenticação que envia as credenciais do usuário em formato de texto simples. Podemos realizar autenticação básica em uma solicitação utilizando o RestAssured da seguinte forma:

given()
    .auth()
    .basic("username", "password")
    .when()
    .get("/api/protected")
    .then()
    .statusCode(200);

2.2. Autenticação com Token

Em alguns casos, é necessário utilizar tokens de acesso para autenticar as solicitações. O RestAssured permite adicionar esse tipo de autenticação facilmente. Veja um exemplo de como fazer isso:

given()
    .header("Authorization", "Bearer <access_token>")
    .when()
    .get("/api/protected")
    .then()
    .statusCode(200);

3. Testando Requisições Assíncronas

É comum encontrarmos APIs que trabalham com requisições assíncronas, como por exemplo, a criação de recursos com um processo de processamento em segundo plano. O RestAssured permite testar essas APIs de maneira eficiente. Veja como podemos fazer isso:

Response response = given()
    .when()
    .post("/api/resource")
    .then()
    .statusCode(202)
    .extract().response();

// Aguardar um tempo para execução do processo

given()
    .when()
    .get(response.getHeader("Location"))
    .then()
    .statusCode(200)
    .body("status", equalTo("completed"));

4. Especificando Cabeçalhos e Parâmetros

O RestAssured nos oferece recursos para especificar cabeçalhos e parâmetros nas solicitações. Isso é útil quando precisamos enviar dados extras para a API que estamos testando. Veja alguns exemplos:

4.1. Enviando Cabeçalhos

given()
    .header("Content-Type", "application/json")
    .header("Accept", "application/json")
    .when()
    .get("/api/resource")
    .then()
    .statusCode(200);

4.2. Enviando Parâmetros

given()
    .param("query", "example")
    .param("filter", "true")
    .when()
    .get("/api/resource")
    .then()
    .statusCode(200);
